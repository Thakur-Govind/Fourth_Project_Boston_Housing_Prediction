Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Question 3 - Training and Testing¶

  • What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?

Hint: Think about how overfitting or underfitting is contingent upon how splits on data is done.

Answer:

Whenever we design a model for a dataset, we must check how useful the model is. The model should not classify data by too simple a separation (underfitting), nor should it be too specific in its classification (overfitting). The model should work in such a way, that through it we are able to derive a generalised trend, which has high accuracy, to help make predictions easier.

As a result, we should set aside some part of the data we have, just to check if the model is useful or not. The set aside data is the testing subset. Using this subset, we can check how the model works on this subset, and see if the resultant classification is correct or not. We cant expect it to be 100% accurate, but the classification should be nearly 100% accurate.

The ratio in which the dataset is split is also important. We should not use a large percentage of the data as the testing subset. Nor should 100% of the data be used as the testing subset. The reason is, that once this testing subset is selected, it can never be touched. Also, if we keep too high a ratio of test:train, the correct model might not be able to classify the training data perfectly, resulting in rejection of that model, due to its mismatch with the testing subset.

Answer:

Models have 2 types of parameters. One type is the ones the model learns, and the other type is specified by us, called hyperparameters.

Some models, at times, have more than 1 hyperparameter. Since these are defined by us, we will assign random values to them. Hence, it becomes difficult to chose a good combination of these parameters, for making an efficient model, which properly classifies data. This is where the Grid Search Technique comes into play.

We input possible values for each hyperparameter, and a performance metric to judge the effectiveness of a combination of hyperparameters. The algorithm arranges all of the possible combinations into a grid, and with the help of the performance metric we specified, finds out which combination is the best for our dataset.

Finally, the output is the hyper-parameter combination that gives the best result.

The performance metric is really important for the algorithm to function. It is on the basis of the performance metric, that each combination is scored, and the best combination is found out.

Question 8 - Cross-Validation¶

  • What is the k-fold cross-validation training technique?

  • What benefit does this technique provide for grid search when optimizing a model?

Hint: When explaining the k-fold cross validation technique, be sure to touch upon what 'k' is, how the dataset is split into different parts for training and testing and the number of times it is run based on the 'k' value.

When thinking about how k-fold cross validation helps grid search, think about the main drawbacks of grid search which are hinged upon using a particular subset of data for training or testing and how k-fold cv could help alleviate that. You can refer to the docs for your answer.

Answer:

As a rule, we cannot access our testing subset once it has been intialised. But that may lead to loss of data, as the testing set could have data, that should have been used in the training data. This leads to variance

The solution to this problem is the K-Fold Cross-Validation . In K-Fold Cross-Validation, the training data is divided into K folds. Out of the K folds, K-1 sets are used for training while the remaining set is used for testing. The algorithm is trained and tested K times, each time a new set is used as testing set while remaining sets are used for training. Finally, the result of the K-Fold Cross-Validation is the average of the results obtained on each set.

In the k-fold cross validation method, all the entries in the original training data set are used for both training as well as validation. Also, each entry is used for validation just once.

For example, let us consider a dataset to be of size 8, with the testing subset of size 2.

Let the data be [1,2,3,4,5,6,7,8]. As the size of the testing subest is 2, the model is split 4 times. This is how the process undergoes(left-training, right-testing) :

[3,4,5,6,7,8] , [1,2]

[1,2,5,6,7,8] , [3,4]

[1,2,3,4,7,8] , [5,6]

[1,2,3,4,5,6] , [7,8]

Now, the average value of all fields is taken. Using standard deviation of the results obtained from each fold we can in fact find the variance in the overall result. Using normal cross-validation also has its disadvantages, since you use up a part of your training data. Hence, here k-fold comes to the rescue.

The K fold cross validation can be used for finding and improving the performance of a model, by reducing the error due to variance. The K fold cross valitdation can give better entries in the grid, helping to easily identify which model has a good score, and which doesn't

from sklearn.metrics import make_scorer
from sklearn.tree import DecisionTreeRegressor
from sklearn.grid_search import GridSearchCV
def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    # sklearn version 0.18: ShuffleSplit(n_splits=10, test_size=0.1, train_size=None, random_state=None)
    # sklearn versiin 0.17: ShuffleSplit(n, n_iter=10, test_size=0.1, train_size=None, random_state=None)
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth':np.arange(1,11)}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(regressor, params, scoring=scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)
    
    # Return the optimal model after fitting the data
    return grid.best_estimator_

Answer:

The range obtained is about 69,000, a high value.

Despite the model having such a good prediction system, it should not be set in a real-world setting.

First of all, data. If the model is implemented in a real-world setting, the data size will be at least 10000 times it is now. Having so much data also increases the possibility of errors in prediction.

The price of a particular property does not remain constant for its lifetime. Data collected in 1978 not not be applicable in the year 2019. According to investopedia,

  • "Inflation is a quantitative measure of the rate at which the average price level of a basket of selected goods and services in an economy increases over a period of time. It is the constant rise in the general level of prices where a unit of currency buys less than it did in prior periods. Often expressed as a percentage, inflation indicates a decrease in the purchasing power of a nations currency".

Inflation is very important, and should be taken into consideration. The model does not have Inflation as a variable.

Talking about variables, only the three factors (rooms, poverty rate, student-teacher ratio) cannot effectively decide how costly a house may be. There are several other examples.

For instance, the quality and amount of furniture also decides the cost of the house.

Other points are location(proximity to shoping malls, train stations, airport, offices), safety in the area (both structural, and societal), water supply, power supply, quality of emergency systems, availability of parking and pools, etc Obviously, not all places have the same enviroment as Boston. The model should train the data to be place-specific too.

This model does not take all of the above points into account.

Hence, this model should not be used in a world wide approach.
